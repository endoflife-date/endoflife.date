**Cisco NX-OS** is a data center network operating system that runs on Cisco Nexus-series Ethernet data switches and MDS-series storage switches. It originated from Cisco’s SAN-OS and evolved from the Cisco IOS code base into a modular OS supporting high-performance switching features for modern data centers. Cisco NX-OS can operate in two modes: **NX-OS standalone mode** (common on Nexus 3000–9000 series switches) and **ACI mode**, where Nexus 9000 switches are managed as an Application Centric Infrastructure (ACI) fabric via Cisco APIC controllers.

The table below lists the lifecycle of recent **Cisco NX-OS** release **trains**, including both NX-OS mode and ACI mode versions. Each release train’s *Maintenance Support* date corresponds to the **End of Software Maintenance** milestone (after which only security fixes are provided), the *Security Support* date corresponds to the **End of Vulnerability/Security Support** milestone (after which only limited technical service may be available), and the *Service Support* date corresponds to the **Last Date of Support** milestone (after which all support for that release ends). Cisco’s standard policy provides around 30 months of maintenance and ~54 months of security support for NX-OS mode trains, with a final service support date about 6–9 months later; and for ACI mode releases, around 27 months of maintenance and ~42 months of security support, with a final service support date roughly 12 months later (some variations apply to long-term support releases). Always refer to official **Cisco NX-OS** end-of-life bulletins or the Cisco **NX-OS Software Lifecycle Policy** for authoritative support dates.
